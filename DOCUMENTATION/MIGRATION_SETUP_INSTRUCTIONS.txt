ERROR HANDLING & ROLLBACK
================================================================

If migration fails:

OPTION 1: Rollback to previous migration
---------
dotnet ef database update [PreviousMigrationName]

Example:
dotnet ef database update 20240101120000_PreviousMigration

OPTION 2: Remove failed migration
---------
dotnet ef migrations remove

Then fix issues and retry:
dotnet ef migrations add AddUserSettingsTables

OPTION 3: Manual SQL Rollback
---------
Run reverse SQL to drop tables:
DROP TABLE IF EXISTS "UserNotificationSettings" CASCADE;
DROP TABLE IF EXISTS "UserPrivacySettings" CASCADE;
DROP TABLE IF EXISTS "UserTwoFactorAuthentication" CASCADE;
DROP TABLE IF EXISTS "UserSessions" CASCADE;

Then retry migration from STEP 4.
